# 美团外卖评分体系看板调整方案

> 编制日期：2026-07-17 | 版本：v1.0 | 基于美团 2025.10 + 2026.06 评分规则

---

## 一、评分体系变化总览

### 1.1 店铺分（Store Score）—— 2026 年 6 月下旬，已全国推行

| 对比维度 | 旧规则 | 新规则（2026.06） |
|---------|--------|------------------|
| 体系架构 | 装修丰富度/菜品丰富度/服务功能丰富度等碎片化指标 | **资质、营业、信息、服务四大维度，9 个指标** |
| 满分 | 100 分 | 100 分（不变） |
| 食品安全 | 无专项指标 | **资质核验 10 分 + 后厨环境核验 10 分 = 20% 权重** |
| 已下线指标 | — | 差评回复率、商家不接单率、在线联系回复（重复性指标） |
| 零分惩罚 | 无 | 180 天内未完成门店验真 → 资质核验 0 分；未开通后厨直播 → 环境核验 0 分 |
| 评分导向 | 运营丰富度 | **食品安全底线 + 合规经营 + 服务能力** |

> 来源：央广网/北京商报/新京报 2026-06-09 联合报道

### 1.2 综合评价分（Comprehensive Rating）—— 2025 年 10 月，已全国推行

| 对比维度 | 旧规则 | 新规则（2025.10） |
|---------|--------|------------------|
| 计算方式 | 近 30 天用户主观打分均值 | **六因子加权综合评分** |
| 商品质量分 | — | 商品满意度 30% + 包装满意度 10% + 复购率 20% + 食安负反馈率 20% |
| 服务体验分 | — | 消息回复率 10% + 服务负反馈率 10% |
| 计算周期 | 近 30 日 | 满意度类近 60 日 / 行为类近 30 日 |
| 新商家保护 | 无 | 有效评价积累到一定数量前不展示评分 |
| 恶意差评治理 | 每周 3 次申诉 | 无限次申诉 + AI 自动拦截 + 前置豁免 |

> 来源：美团外卖平台评价管理规范（rules-center.meituan.com/rule-detail/3469）

## 二、当前看板与最新规则的差距分析

### 2.1 店铺分模块

| # | 问题 | 影响范围 | 严重度 |

| # | 问题 | 影响范围 | 严重度 |
|---|------|---------|--------|

| 2 | 商品质量分、服务体验分两个聚合指标数据有但 UI 不展示 | 概览树 | 🟢 低 |
| 3 | 计算周期未标注（60 日 vs 30 日） | DIM_META | 🟢 低 |

## 三、调整方案详析

### 3.1 店铺分模块

#### 3.1.1 数据处理层（gen_data_v4.py）

| 操作 | 文件位置 | 具体修改 |
|------|---------|---------|
| **删除** | dim_labels (L336) | 删除 `'reject_rate': '商家不接单率得分'`, `'reply_rate': '差评回复率得分'` |
| **新增** | dim_labels | 添加 `'cert_verify': '资质核验得分'`, `'kitchen_env': '后厨环境核验得分'` |
| **删除** | shop_dims 构建 (L400-427) | 删除 `reject_rate` 和 `reply_rate` 的取值逻辑 |
| **新增** | shop_dims 构建 | 添加 `cert_verify` → 读取 Excel 列 `资质核验得分`（如无此列，暂设为 0 并标注"数据源待补充"）|
| **新增** | shop_dims 构建 | 添加 `kitchen_env` → 读取 Excel 列 `后厨环境核验得分`（同上处理） |
| **修改** | 预警维度列表 (L800-801) | 10 个维度 → 9 个维度：删除 reply_rate/reject_rate，添加 cert_verify/kitchen_env |
| **保持** | 预警阈值 shop_score < 95 | 不变 |
| **保持** | mt_daily 汇总逻辑 | 不变 |

#### 3.1.2 前端展示层（dashboard/index.html）

**A. 概览区评分树（优先级：P0）**

位置：`calcRealMetrics()` → `mtShopBlock.dims`（L4512-4517）

删除 "差评回复率得分" 和 "商家不接单率" 两个维度卡片；在信息维度区新增 "资质核验" 和 "后厨环境核验"。

> 建议维度排列顺序（按新规则四大维度分组）：

```
┌─ 资质 ─────────────────────────────┐
│ 资质核验      当前:--  目标:100  10分 │
├─ 营业 ─────────────────────────────┤
│ 基础营业时长   当前:95  目标:100      │
│ 高峰营业时长   当前:90  目标:100      │
│ 出餐上报/准时率 当前:88  目标:100     │
├─ 信息 ─────────────────────────────┤
│ 后厨环境核验   当前:--  目标:100  10分 │
│ 菜单丰富度     当前:85  目标:100      │
│ 装修丰富度     当前:92  目标:100      │
├─ 服务 ─────────────────────────────┤
│ 商家评分       当前:90  目标:100      │
│ 优质商品率     当前:95  目标:100      │
│ 服务功能丰富度  当前:80  目标:100      │
└────────────────────────────────────┘
```

**B. 门店详情抽屉（优先级：P0）**

位置：`renderMtDetail()` (L3808-3835) 和双平台抽屉 (L3623-3651)

- 删除 `dimRow('差评回复率得分', ...)` 和 `dimRow('商家不接单率', ...)`
- 新增 `dimRow('资质核验', sd.cert_verify, 100, 100, '分')`
- 新增 `dimRow('后厨环境核验', sd.kitchen_env, 100, 100, '分')`
- 标题从"六维明细"改为"九维明细"

**C. DIM_META 释义库（优先级：P0）**

新增两个条目：

```javascript
cert_verify: {
  cat:'shop', field:'资质核验得分',
  def:'门店真实性验证，包括营业执照、经营地址、门面照等资质材料的实地比对',
  calc:'上传"一镜到底"门头到后厨视频并完成交叉验证，180天内未完成→0分',
  source:'平台审核系统', tips:'尽快完成门店验真，联系区域经理协助拍摄',
  target:100, max:100
},
kitchen_env: {
  cat:'shop', field:'后厨环境核验得分',
  def:'后厨环境卫生达标验证',
  calc:'开通后厨直播并保持7日内在播，或完成平台环境核验任务，未完成→0分',
  source:'后厨直播系统', tips:'开通后厨直播功能，确保连续7日在播',
  target:100, max:100
}
```

更新 `dimActions` 和 `dimTips` 中对应条目的改善建议。

删除 DIM_META 中 `reply_rate` 和 `reject_rate` 条目。

**D. 预警模块（优先级：P1）**

- `typeLabelMap` / `labelMap`：店铺分预警标签文本保持不变（"店铺分预警"）
- 预警详情弹窗 `openAlertDetail()`：维度阈值判定列表自动跟随 gen_data_v4.py 输出的新 9 维数据
- 预警详情弹窗中 `detail` 字段解析：`dim_row.py` 中 `dim_labels` 已更新，detail 文本自动适配

#### 3.1.3 数据源兼容性处理（优先级：P0）

**关键风险**：当前数据源（7 月 Excel）的「美团指标数据源」sheet 中**可能不存在**"资质核验得分"和"后厨环境核验得分"列。需要在 gen_data_v4.py 中添加兼容逻辑：

```python
# 兼容处理：新指标列不存在时标记为缺失
cert_col = '资质核验得分'
kitchen_col = '后厨环境核验得分'

if cert_col in mt_header_idx:
    cert_verify = safe_float(col_val(row, mt_header_idx, cert_col))
else:
    cert_verify = None  # 前端展示为"--"或"待补充"
```

### 3.2 综合评价分模块

**总体判断**：综合评价分的数据结构已匹配新规则，调整重点是**信息展示层的补充**，不涉及数据处理层修改。

#### 3.2.1 权重标注（优先级：P1）

在 DIM_META 和门店详情抽屉中为每个因子标注权重：

| 因子 | 权重 | 所属维度 | 计算周期 |
|------|------|---------|---------|
| 商品满意度 | **30%** | 商品质量分 (80%) | 近 60 日 |
| 包装满意度 | **10%** | 商品质量分 (80%) | 近 60 日 |
| 复购率 | **20%** | 商品质量分 (80%) | 近 30 日 |
| 食品安全负反馈率 | **20%** | 商品质量分 (80%) | 近 30 日 |
| 消息回复率 | **10%** | 服务体验分 (20%) | 近 30 日 |
| 服务负反馈率 | **10%** | 服务体验分 (20%) | 近 30 日 |

#### 3.2.2 聚合指标展示（优先级：P2）

在概览区评分树下添加 `商品质量分` 和 `服务体验分` 两个聚合指标（数据源中已有），作为对应原子因子的分组标题行。

#### 3.2.3 DIM_META 更新

为每个因子补充权重和计算周期：

```javascript
product_sat: {
  cat:'exp', field:'商品满意度', weight:'30%', period:'近60日',
  def:'近60日用户对商品满意度项打分的算术平均值',
  ...
}
```

## 七、总结

| 维度 | 变化量 | 核心影响 |
|------|--------|---------|
| 店铺分 | 删除 2 个旧指标、新增 2 个食安指标、UI 全面重构 | 当前看板展示的 2 个指标已无效，2 个食安指标完全缺失 |
| 综合评价分 | 数据结构匹配，主要是展示层优化（权重、周期标注） | 商家无法了解各因子对总分的贡献程度 |
| 预警 | 维度列表更新，阈值保持不变 | 新指标加入后预警更精准 |
| 风险 | 数据源可能缺失新指标列 | 需兼容处理，前端展示"待补充" |

**建议实施路径**：先行确认事项 #1（数据源）→ 执行 P0 调整 → 部署上线 → 两周内完成 P1 优化。
