## 二、发现的问题

### 🔴 P0 — 致命级

#### 问题1：小程序配送数据(MP)遗漏33天OLD数据

**位置**：`gen_data.py` 第267行

```python
df2 = pd.read_excel(NEW_PATH, sheet_name='小程序配送数据源')
```

**事实**：只读取了NEW文件，遗漏OLD文件中的33天MP数据（630,076行）。

**连锁影响**：

| 受影响的数据项 | 当前值 | 应有值 | 偏差 |
|-------------|--------|--------|------|
| mp_daily | 30天 | 63天 | 缺55%数据 |
| store_daily.mp覆盖率 | 30天 | 63天 | MP侧门店日期筛选不完整 |
| mp_stores | 仅近7天 | 全量(无日期维度) | — |
| 出餐超时预警 | 只检测6月 | 5-7月全量 | 漏检5月超时 |
| mp_breakdown/热力图 | 仅7天 | 仅7天(本身设计如此) | — |
| mp_summary达标率 | 仅NEW数据 | 全量 | 偏差取决于两批数据差异 |

**对比**：美团指标数据(MT)在第164-166行正确使用了`pd.concat([mt_old, mt_new])`。

#### 问题2：perf_daily & order_daily 始终为空

**位置**：`gen_data.py` 第546行

```python
perf_daily_list = []; order_daily_list = []; full_date_range = ''
```

**事实**：这两个列表被声明后从未被填充。

**连锁影响**：
- 前端 `PERF_DAILY` 和 `ORDER_DAILY` 始终为空数组
- `calcRealMetrics()` 中的 `mpRevTotal` 和 `mtRevTotal` 全部为0
- **「外卖营收」指标完全失效**
- 前端第4176-4191行的业绩推算逻辑(line 4176-4191)因perfDayFiltered为空永远走else分支

---

### 🟡 P1 — 严重级

#### 问题3：market/brand/region排行榜不随日期筛选变化

**位置**：`index.html` `renderMarketBrandRank()` (3759行)、`renderRegionMgrRank()` (3848行)、`renderAreaMgrRank()` (3922行)

**事实**：这些函数直接使用 `filterStores(STORE_MP)` / `filterStores(STORE_MT)` 聚合数据，没有调用 `getDateFilteredStoreCodes()` 进行日期过滤。

**影响**：用户切换日期范围时，排行榜数据不变。

#### 问题4：MT门店列表仅取"最新单日"

**位置**：`gen_data.py` 第193行

```python
mt_latest = mt[mt['日期'] == latest].copy()
```

**事实**：`mt_stores` 只包含最新一天有数据的门店。

**影响**：如果在最新日期某门店没有美团数据（如当天未开平台），该店在排行榜中消失。MT侧缺少类似 `mp_stores` 的滑动窗口设计。

#### 问题5：MP门店列表仅取"近7日"

**位置**：`gen_data.py` 第270行

```python
df2_7d = df2[df2['日期_dt'] >= max_date - pd.Timedelta(days=6)].copy()
```

**事实**：`mp_stores` 只包含最近7天内有订单的门店。

**影响**：7天内无单的门店从排行榜消失，且与MT侧(单日)口径不一致。

---

### 🟢 P2 — 优化级

#### 问题6：mp_summary_full 数据不完整

**位置**：`gen_data.py` 第623行

**事实**：`mp_summary_full` 只有 `total_orders` 一个字段，且基于缺失OLD数据的 `df2`。

**影响**：前端展示的"全量订单数"不包含5月数据。

#### 问题7：mp_summary vs mp_summary_full 口径歧义

- `mp_summary`：基于 `df2_valid`（过滤7-24时段）的7天数据，含 `avg_cook_time`、`达标率`
- `mp_summary_full`：基于 `df2`（未过滤时段），但只有 `total_orders`

两者名称暗示"全量/汇总"但实际口径不一致。

#### 问题8：昼夜诗品牌名不一致

**位置**：`gen_data.py` 输出 '昼夜诗'，`index.html` 第5325行硬编码映射为 '昼夜'

**事实**：`applyRealData()` 中有 `brand: s.brand === '昼夜诗' ? '昼夜' : s.brand`。

**影响**：暂无大问题，但为日后埋坑——gen_data.py改品牌名时可能漏改这里。
